**1. Create tests/e2e/child-dashboard.spec.ts**:

Test setup:
- Use the same seed/cleanup pattern from `tests/e2e/child-login.spec.ts`
- Create a test child account in `test.beforeAll` using adminClient
- The child should have some progress_entries pre-seeded (insert 3-5 progress entries with known points_earned values so we can verify the sum)
- Clean up in `test.afterAll`

Test cases:
- **"dashboard shows greeting with display name"**: Login as child, navigate to /kind/dashboard, verify `page.getByText(/Hallo/)` is visible
- **"dashboard shows grade level"**: Verify `page.getByText(/Klasse \d/)` is visible
- **"dashboard shows total points"**: Sum up the seeded points_earned values, verify that sum appears on the page (e.g., `page.getByText(/30 Punkte/)` if 3 correct easy answers = 30 points)
- **"dashboard shows exercise count"**: Verify the seeded exercise count appears
- **"dashboard shows progress bar"**: Verify `page.getByText(/Punkte bis zum Spiel/)` is visible
- **"Aufgaben starten button links to ueben"**: Verify `page.getByRole('link', { name: /Aufgaben starten/ })` has href containing `/kind/ueben`

**2. Create tests/e2e/exercise-session.spec.ts**:

Test setup:
- Same seed pattern -- create test child, NO progress entries needed (start fresh)
- Login as child before each test

Test cases:
- **"exercise session loads and shows an exercise"**: Navigate to /kind/ueben, wait for exercise display (text matching pattern like `\d+ [+\-*/] \d+ =`), verify exercise text is visible in large font
- **"number pad buttons are visible and clickable"**: Verify buttons 0-9, delete (arrow), and OK are visible. Click "5", verify "5" appears in the answer display area.
- **"submitting correct answer shows green feedback"**: This test needs to be carefully constructed:
  1. Read the exercise from the page (parse operand1, operator, operand2)
  2. Compute the correct answer in the test
  3. Type the correct answer digit by digit using number pad buttons
  4. Click OK/confirm
  5. Wait for feedback: verify text matching `Richtig! \+\d+ Punkte` appears
  6. Verify green background class is applied
- **"submitting wrong answer shows red feedback"**: 
  1. Read exercise, compute correct answer, submit a wrong answer (correct + 1, or just "99999")
  2. Verify text matching `Leider falsch. Die Antwort ist: \d+` appears
  3. Verify red background class is applied
- **"feedback auto-advances to next exercise"**: After submitting an answer, wait ~2.5 seconds, verify a new exercise is displayed (the text changes)
- **"Beenden button returns to dashboard"**: Click "Beenden" button, verify URL changes to /kind/dashboard
- **"session stats update after answering"**: After answering one exercise, verify "1 richtig" or "1 falsch" text appears in stats area

**Important test implementation details:**
- Use `page.waitForSelector` or `page.waitForTimeout` where async state transitions happen
- Use `test.slow()` for tests involving feedback timers (2+ seconds wait)
- For exercise parsing: use `page.textContent()` on the exercise display element and regex to extract operand1, operator, operand2
- For computing answers in test: simple switch on operator (+, -, *, /)
- The E2E test must handle the case where the exercise involves large numbers -- type each digit individually via number pad
