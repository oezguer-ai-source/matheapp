**1. Create components/child/exercise-session.tsx** (per D-06 through D-15, D-22 through D-24):

This is the main Client Component with a state machine. "use client" directive.

**Props:** `{ grade: number }`

**State machine states:** `"loading" | "answering" | "submitting" | "feedback" | "error"`

**State variables (all useState):**
- `state: SessionState` -- starts at "loading"
- `exercise: ClientExercise | null` -- current exercise
- `answer: string` -- digit buffer for typed answer (starts "")
- `difficulty: Difficulty` -- starts "easy" (per D-23)
- `correctStreak: number` -- starts 0 (per D-23)
- `incorrectStreak: number` -- starts 0 (per D-23)
- `feedback: SubmitAnswerResult | null` -- last submit result
- `correctCount: number` -- session stats counter (per D-15)
- `wrongCount: number` -- session stats counter (per D-15)

**useRef:** `timeoutRef = useRef<ReturnType<typeof setTimeout> | null>(null)` for feedback timer cleanup (Pitfall 6)

**useEffect on mount:** Call `loadNextExercise()` once. Cleanup function clears `timeoutRef.current`.

**loadNextExercise() function:**
1. Set state to "loading"
2. Call `generateExerciseAction(grade, difficulty)`
3. If result.data: set exercise, clear answer to "", set state to "answering"
4. If result.error: set state to "error"

**Number pad handlers:**
- `handleDigit(digit)`: Append digit to answer string. **Limit to 5 digits max** (Pitfall 4: max valid answer for grade 4 hard is ~10000). Ignore if answer.length >= 5.
- `handleDelete()`: Remove last character from answer. `setAnswer(prev => prev.slice(0, -1))`
- `handleConfirm()`: If answer is empty or state !== "answering", do nothing. Otherwise call `handleSubmit()`.

**handleSubmit() function** (per D-22):
1. Set state to "submitting"
2. Call `submitAnswerAction({ exerciseId: exercise.id, operand1: exercise.operand1, operand2: exercise.operand2, operator: exercise.operator, userAnswer: parseInt(answer, 10), currentDifficulty: difficulty, correctStreak, incorrectStreak })`
3. If result.data:
   - Set feedback to result.data
   - Update streaks: `setCorrectStreak(result.data.newCorrectStreak)`, `setIncorrectStreak(result.data.newIncorrectStreak)`
   - Update difficulty: `setDifficulty(result.data.newDifficulty)`
   - Update counters: if correct `setCorrectCount(c => c + 1)` else `setWrongCount(c => c + 1)`
   - Set state to "feedback"
   - Auto-advance timer (per D-14): `timeoutRef.current = setTimeout(() => loadNextExercise(), result.data.correct ? 1500 : 2000)`
4. If result.error: set state to "error"

**handleEndSession() function** (per D-11, D-24):
1. Clear any pending timeout: `if (timeoutRef.current) clearTimeout(timeoutRef.current)`
2. `router.push("/kind/dashboard")`
3. `router.refresh()` -- re-fetches Server Component data to show updated points

**Render (structured by state):**

Outer container: `min-h-dvh transition-colors duration-300` with conditional background:
- feedback + correct: `bg-green-50`
- feedback + not correct: `bg-red-50`
- otherwise: `bg-white`

**Header bar (always visible per D-11, D-15):**
- "Beenden" button: top-left, `bg-child-red text-white rounded-2xl px-6 h-12 text-xl font-semibold`. Calls `handleEndSession()`.
- Session stats (per D-15): top-right area showing:
  - Correct count with green dot: `{correctCount} richtig`
  - Wrong count with red dot: `{wrongCount} falsch`
  - Current difficulty: display as German label (easy="Leicht", medium="Mittel", hard="Schwer")

**Main content area (centered):**

When state === "loading": Show loading indicator (e.g., `"Aufgabe wird geladen..."` in text-2xl)

When state === "answering" or "submitting":
- Exercise display (per D-07): `"{exercise.operand1} {exercise.operator} {exercise.operand2} = ?"` in `text-5xl font-bold text-center` with the "?" replaced by the current answer string if answer is not empty
- Answer display (per D-10): Show typed digits large (text-5xl) in place of the "?" -- real-time update as child taps digits. If answer is empty show "?" as placeholder.
- NumberPad component below exercise (per D-08, D-09): disabled during "submitting" state (Pitfall 2)

When state === "feedback":
- FeedbackOverlay component with correct/pointsEarned/correctAnswer from feedback state
- NumberPad still visible but disabled

When state === "error":
- Error message with "Erneut versuchen" button that calls `loadNextExercise()`

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client Component -> Server Actions | User input (answer digits) crosses trust boundary via submitAnswerAction |
| Browser state -> Server | Difficulty/streak sent from client; server validates operands against grade |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-30-03 | Tampering | Number pad answer manipulation | mitigate | Server re-computes correct answer from operands (Phase 20 Pattern 3); client never knows answer before submission. Zod validates input shape server-side. |
| T-30-04 | Tampering | Rapid duplicate submissions | mitigate | Disable confirm button during "submitting" state (Pitfall 2). Server writes are idempotent per exercise_id. |
| T-30-05 | Tampering | Difficulty/streak client state forgery | accept | Client sends current difficulty and streaks. Server validates operands against grade/difficulty range (validateOperandsForGrade in actions.ts). Forging easy difficulty with simple operands still earns only easy-tier points. Risk: low value, effort not worth mitigation. |
| T-30-06 | Denial of Service | setTimeout memory leak | mitigate | Store timeout ID in useRef, clear on unmount and on "Beenden" click (Pitfall 6). |
| T-30-07 | Information Disclosure | Correct answer in client state | mitigate | correctAnswer only stored in feedback state AFTER server responds. Never available before submission. |
</threat_model>
