    4. Create `lib/exercises/generators.ts`:
       - `export function randomInt(min: number, max: number): number` — `Math.floor(Math.random() * (max - min + 1)) + min` (inclusive both ends, per Pitfall 1)
       - `export function compute(a: number, b: number, op: Operator): number` — switch on operator, returns result
       - Internal `pickOperator(grade: Grade, difficulty: Difficulty): Operator` — picks random operator from `RANGES[grade][difficulty].operators`
       - Internal `generateOperands(operator: Operator, config: RangeConfig): { operand1: number; operand2: number }`:
         - For `+` and `*`: both operands in [config.min, config.max]
         - For `-`: generate both in range, then swap if operand1 < operand2 (D-10, Pattern 5)
         - For `/`: use answer-first approach (Pattern 4) — pick divisor in [2, min(config.max, 10)] for grades 1-3, [2, min(config.max, 20)] for grade 4; pick quotient so dividend = quotient * divisor stays within config.max; return { operand1: dividend, operand2: divisor }
       - `export function generateExercise(grade: number, difficulty: Difficulty): Exercise` — validates grade is 1-4, gets config from RANGES, picks operator, generates operands, computes answer, returns `{ id: crypto.randomUUID(), operand1, operand2, operator, correctAnswer }`

       Key implementation details for division (D-09, Pitfall 2, Pitfall 6):
       - Divisor always >= 2 (avoids division by zero and trivial /1)
       - For grade 3: divisor range [config.min, config.max] (max 10 for Einmaleins)
       - For grade 4: divisor capped at 20 regardless of range max
       - quotient = randomInt(1, Math.floor(config.max / divisor))
       - operand1 (dividend) = quotient * divisor — guarantees whole number result
